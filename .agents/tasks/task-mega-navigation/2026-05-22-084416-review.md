# Premium mega navigation section for Dawn (v2)

All seven issues from the v1 review have been addressed. Mobile drawer now renders all 6 columns. `aria-expanded` is managed on both the hamburger button and accordion triggers. A focus trap locks Tab within the open drawer. The sticky offset recalculates on every scroll. Arrow-key navigation (Left/Right between items, Down into dropdown) is implemented. Promo image carries `width="560" height="560"`. The `mn_main_menu` omission is acknowledged as a deliberate blocks-only design choice.

**Watch for:** Sticky recalculation calls `getBoundingClientRect` on every scroll frame which will thrash layout if the section has complex paint (likely); mobile drawer lacks `role="dialog"` and `aria-modal="true"` for full WCAG 2.1 dialog conformance (confirmed); mobile link titles render `[SALE]`/`[NEW]`/`[HOT]` as literal text instead of styled badges (confirmed).

**Verdict**: APPROVED

## High-level view

The sticky nav fix replaces the one-time offset capture with a per-scroll `getBoundingClientRect` call. This introduces forced synchronous layout on every scroll frame — negligible for a simple nav bar, but a pattern that doesn't scale to complex page layouts without throttling or switching to IntersectionObserver.

The remaining accessibility gap is that the drawer element doesn't declare `role="dialog"` or `aria-modal="true"`, which some screen readers use to suppress the background virtual buffer. The focus trap, `aria-expanded` management, and arrow-key navigation are all correctly wired.

The mobile drawer now includes all 6 column slots with inline `linklists[handle].links` lookups. The inline approach skips badge detection logic that the `mega-nav-column` snippet handles on desktop, so mobile users see literal `[SALE]` text rather than styled badge spans.

## Forced reflow in scroll handler

```javascript
function getOffsetTop() {
  return section.getBoundingClientRect().top + window.pageYOffset;
}

function onScroll() {
  var offsetTop = getOffsetTop();
  ...
}
```

Every scroll event forces the browser to compute layout to resolve `getBoundingClientRect`. On a page with elements above the nav that repaint (animated banners, lazy images, expanding accordions), this becomes a layout-thrash hot path. IntersectionObserver would achieve the same dynamic sticky behavior without per-frame layout cost.

## Missing dialog semantics on mobile drawer

The container `<div class="mega-nav-drawer">` carries no ARIA role. Adding `role="dialog"` and `aria-modal="true"` when open (and removing them when closed) would satisfy the WAI-ARIA Authoring Practices modal dialog requirements. Without it, screen readers may still expose the background page content in the virtual buffer even though keyboard focus is trapped.

## Mobile badge detection gap

Desktop columns render via the `mega-nav-column` snippet which strips `[NEW]`, `[SALE]`, `[HOT]` tags from link titles and renders colored badge spans. The mobile drawer renders link titles directly:

```liquid
{% for link in mobile_links_1 %}
  <li><a href="{{ link.url }}" class="mega-nav-mobile-submenu__link">{{ link.title }}</a></li>
{% endfor %}
```

If a merchant uses the `[SALE]` convention in link titles, mobile users see the literal text `[SALE]` in the link rather than a styled badge. Either call the snippet for mobile columns too, or inline the badge-strip logic in the mobile accordion loop.
