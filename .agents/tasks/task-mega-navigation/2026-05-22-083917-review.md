# Premium mega navigation section for Dawn

A standalone Shopify section delivering a full-width multi-column mega navigation with per-item color overrides, hover underline animations, badge detection from link titles, promotional cards in dropdowns, and a mobile drawer with accordion sub-menus. The implementation is five new files: a section, a stylesheet, a JS IIFE, and two render snippets. The approach uses blocks for menu items (each with up to 6 column link-list pickers), CSS custom properties for all 50 configurable settings, and `linklists[handle]` lookups inside snippets to stay within Shopify's `{% render %}` primitive-passing rules.

**Watch for:** Mobile drawer only shows columns 1-3 while desktop shows all 6 (confirmed); sticky nav offset is computed once on init and never recomputed on layout shift (confirmed); no `aria-expanded` state management on accordion buttons or dropdown triggers (confirmed); `mn_main_menu` setting referenced in the spec but absent from the schema and template (confirmed).

## High-level view

The section is self-contained: CSS scoped to `.mega-navigation-section`, JS in an IIFE with no globals, schema carrying all settings. Two snippets (`mega-nav-column` and `mega-nav-promo`) keep the template readable while complying with Shopify's render-only-primitives constraint.

The desktop experience is hover-driven with configurable open/close delays and four dropdown animation styles. The mobile experience collapses into a slide-from-side or fullscreen drawer with accordion panels, though it silently drops columns 4-6. Keyboard support is limited to Escape-to-close; there is no arrow-key navigation or focus trapping despite the spec calling for both.

The sticky behavior uses a passive scroll listener but bakes in the offset at initialization time, which will misfire after dynamic content above the nav loads or expands.

## Mobile drawer drops columns 4-6

The desktop dropdown renders all six column slots via the `mega-nav-column` snippet. The mobile drawer's inline accordion only checks columns 1-3:

```liquid
{% if block.settings.mn_col1_linklist != blank %}
  ...
{% endif %}
{% if block.settings.mn_col2_linklist != blank %}
  ...
{% endif %}
{% if block.settings.mn_col3_linklist != blank %}
  ...
{% endif %}
{%- comment -%} col4, col5, col6: absent {%- endcomment -%}
```

A merchant configuring a 5-column dropdown will see all links on desktop but only three columns on mobile. This is unlikely intentional given the schema allows `mn_columns` up to 6.

## Missing `mn_main_menu` setting

The spec (FEAT-001 step 6) lists `mn_main_menu (link_list picker)` as a section-level setting. The implemented schema omits it — navigation items are exclusively manual block entries. The blocks-only approach gives per-item control, but it means the spec criterion is unmet and a merchant must manually re-create their entire menu structure as blocks rather than pointing to an existing Shopify menu.

## Accessibility gaps: `aria-expanded`, focus trap, arrow keys

The hamburger button (`<button class="mega-nav-hamburger" aria-label="Open menu">`) never receives `aria-expanded`. The accordion triggers (`<button class="mega-nav-mobile-link" data-has-submenu="true">`) also lack it. When the mobile drawer opens, `document.body.style.overflow = 'hidden'` prevents scroll but nothing traps keyboard focus — a user can Tab out of the drawer into content behind the overlay.

The JS `setupKeyboard` function handles only Escape. The spec's step 4f calls for "arrow keys navigate" which is absent. For a premium component, this leaves WCAG 2.1 Level AA conformance incomplete on the disclosure and menu patterns.

## Sticky offset is static

```javascript
var offsetTop = section.getBoundingClientRect().top + window.pageYOffset;
```

This runs once when `initMegaNavigation` fires. If an announcement bar, cookie banner, or lazy-loaded hero appears above the nav afterward, the trigger threshold is stale. A `ResizeObserver` or `IntersectionObserver` approach would handle dynamic layout changes without recalculating in the scroll handler.

## Promo image layout shift

```liquid
<img src="{{ image }}" alt="{{ image_alt }}" loading="lazy" class="mega-nav-promo__image">
```

No `width` or `height` attributes. For a promotional image inside a dropdown, this causes a visible reflow when the image loads on first hover. Adding explicit dimensions (or an `aspect-ratio` wrapper in CSS) would eliminate the CLS penalty.

## `shopify:block:select` relies on undocumented attribute (possible)

The block select handler queries:

```javascript
var navItem = section.querySelector('[data-block-id="' + block.getAttribute('data-shopify-editor-block') + '"]');
```

`data-shopify-editor-block` is not part of Shopify's public API docs — it's an internal Customizer detail. The risk of breakage is low (Shopify rarely changes editor injection patterns), but the reliance is worth noting for long-term maintenance.
