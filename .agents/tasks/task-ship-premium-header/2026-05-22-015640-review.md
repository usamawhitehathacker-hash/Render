## High-level view

The empty-cart fix closes a Liquid+JS contract: server-side, the
receiver span is always emitted with `aria-hidden="true"` and an inline
`display: none` only when `cart == empty`; client-side,
`syncBottomNavCartCount()` clears that inline `display` on the first
non-empty `.cart-count-bubble` it observes. The empty-cart-then-add path
that v2 flagged is no longer broken because the observer always has a
target to write into.

The push open-state fix scopes the `translateX(0)` rule to non-fullscreen
positions on both selectors. Default, left, and right positions still
get push's X-axis open-state; fullscreen + push now resolves to
fullscreen's `translateY(0)` from the position rule, on the same axis as
the closed `translateY(-100%)`. The unscoped push transition rule above
still applies, so fullscreen + push retains the configured push duration
and easing on the Y-axis slide.

The `hp_cart_badge_override` checkbox reorder is a JSON entry-position
change inside `section.settings`. No setting ids, types, defaults, or
data-attributes changed; only the merchant editing flow ("flip the gate
first, then set colors below") and the `info` string ("set below"
replaces "set above").

The cumulative `7330220..HEAD` diff is a coherent premium-header layer:
47 settings, three production files, master toggle gates the entire
premium surface, every `hp_` id is wired into Liquid + CSS + JS per the
validator. Remaining behavioral side effects (sticky-footer push from
the 64px bottom-nav padding, English wishlist + `<nav aria-label>`) are
documented in schema `info` strings or accepted under the no-new-`t:`-keys
rule.

### Empty-cart receiver span: Liquid + JS contract

The Liquid block now reads:

```liquid
<span
  class="hp-bottom-nav__cart-count"
  aria-hidden="true"
  {%- if cart == empty %} style="display: none;"{%- endif %}
>{%- if cart != empty -%}{{ cart.item_count }}{%- endif -%}</span>
```

`syncBottomNavCartCount()` is the consumer:

```js
var bubble = cartIconBubble.querySelector('.cart-count-bubble span[aria-hidden="true"]');
if (bubble && bubble.textContent) {
  bottomCount.textContent = bubble.textContent.trim();
  bottomCount.style.display = '';   // clears the empty-cart inline display:none
} else {
  bottomCount.style.display = 'none';
}
```

The `aria-hidden="true"` is unconditional, so screen-reader exposure is
unchanged across both paths. The contract is symmetric: Liquid hides
when empty at server render, JS reveals on first non-empty observation
and re-hides when Dawn removes `.cart-count-bubble`.

### Push open-state scoped to non-fullscreen drawer positions

```css
.header-wrapper--premium[data-hp-mobile-anim="push"] .menu-drawer {
  transition: transform var(--hp-anim-duration, 300ms) cubic-bezier(0.4, 0, 0.2, 1);
}
.header-wrapper--premium[data-hp-mobile-anim="push"]:not([data-hp-mobile-drawer="fullscreen"]) menu-drawer[open] .menu-drawer,
.header-wrapper--premium[data-hp-mobile-anim="push"]:not([data-hp-mobile-drawer="fullscreen"]) details[open] > .menu-drawer {
  transform: translateX(0);
}
```

The `:not()` lives on both selectors of the open-state rule, so the
fullscreen + push combination no longer matches the open-state at all
and falls through to fullscreen's position-rule `translateY(0)`. The
push transition rule above stays unscoped so the duration / easing
still apply on fullscreen's Y-axis slide.

### What did NOT change in v3

`assets/header-premium.js`, `validate.py`, the lifecycle observers, the
sticky-footer push side effect, the transparent-header `:has()` fallback,
and the localization story are all unchanged from v2. v3 is a pure
behavior-fix delta on Liquid + CSS, no new surface area.
